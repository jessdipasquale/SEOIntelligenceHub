# Evaluation Report — SEO Intelligence Hub

**Project:** SEO Intelligence Hub
**Evaluator:** LLM-as-Judge (Claude Haiku)

---

## 1. What I Did

For this part of the project I used an LLM-as-judge approach to evaluate 5 insights generated by my SEO gap analysis agent. The idea is simple: instead of comparing the output to a fixed correct answer, I ask another LLM to grade the response based on a set of criteria. This is useful when the output is open-ended and there's no single "right" answer.

I evaluated each insight on four dimensions — relevance, accuracy, actionability, and clarity — each scored from 0 to 1.

---

## 2. Judge Prompt

Here is the prompt I used:
```
You are an expert SEO consultant evaluating AI-generated content gap analysis insights.

You will be given:
- KEYWORD: the target keyword being analyzed
- COMPETITOR_HEADINGS: the top-10 SERP competitor headings used as input
- AGENT_INSIGHT: the gap analysis generated by the AI agent

Evaluate the insight on four criteria, scoring each from 0.0 to 1.0:

1. RELEVANCE: Is the gap relevant to the keyword and what users are searching for?
2. ACCURACY: Is the gap grounded in the competitor headings, without making things up?
3. ACTIONABILITY: Can an SEO analyst act on this right away?
4. CLARITY: Would a non-technical client understand it?

Respond ONLY in this JSON format:
{
  "relevance": 0.0,
  "accuracy": 0.0,
  "actionability": 0.0,
  "clarity": 0.0,
  "overall": 0.0,
  "reasoning": "brief explanation",
  "top_strength": "what works well",
  "top_weakness": "what could be improved"
}
```

I chose these four criteria because they capture what actually matters in an SEO context. Relevance makes sure the AI is solving the right problem. Accuracy is the most important one for me — it directly addresses the risk of hallucination, which is Chleo's main concern. Actionability makes sure the output is useful in practice, not just theoretically interesting. And clarity ensures that even a non-technical client could read the report and understand it.

## 3. Results

### Insight 1 — "best project management software"

The agent identified three gaps: a pricing comparison table, free tier limitations, and integration ecosystem coverage. The quick win was "add a pricing comparison table — present in 8/10 top competitors."

This insight scored well overall (0.93). All three gaps are grounded in the competitor headings and the quick win is specific enough to act on immediately. The integration ecosystem gap is the most nuanced one — it requires understanding that users searching for PM software care about how it fits into their existing tools, not just what it does on its own.

|---|---|
| Relevance | 0.92 |
| Accuracy | 0.95 |
| Actionability | 0.90 |
| Clarity | 0.95 |
| **Overall** | **0.93** |

### Insight 2 — "technical SEO audit"

The agent identified Core Web Vitals (with specific thresholds like LCP <2.5s), structured data, and mobile-first indexing as the main gaps. The quick win was to add a Core Web Vitals section with actual numbers.

This scored 0.90. The relevance is high — these are genuinely important technical SEO topics. The accuracy dipped slightly because the specific metric thresholds (LCP <2.5s, FID <100ms) go beyond what the competitor headings actually showed. The agent added technical knowledge from its training rather than strictly from the input. This is borderline hallucination and something I'd want to fix in the prompt.

The agent identified a Google Business Profile checklist, NAP consistency guide, and local schema markup as gaps. The quick win claimed it was "the most clicked section across competitor pages."

This is exactly the kind of issue that would damage trust with a client like Chleo.

## 5. Bias and Limitations

The biggest issue with this evaluation is that I used Claude Haiku as both the agent and the judge. This is a problem because the judge might unconsciously favour outputs from the same model family. A better approach would be to use GPT-4o or another model as the judge to get a more independent opinion.

I also noticed that all scores fell in a narrow range (0.88–0.94). This could mean the insights are genuinely good, or it could mean the judge is being too generous. Without testing some deliberately bad examples, I can't tell which it is. Adding "negative control" examples to the evaluation dataset would help calibrate this.

Finally, the judge can't verify statistical claims like "9/10 competitors cover this topic." It can only evaluate whether the claim sounds plausible based on the headings provided. For a production system, these claims should always be backed by the actual data.

---

## 6. What I Would Improve

If I were to iterate on this, I would require the agent to cite which specific competitor covers each gap, rather than making aggregate claims. I would also weight accuracy higher than the other criteria, since in an SEO consulting context a wrong recommendation is worse than a vague one. And I would run the evaluation on a larger set of keywords — 5 is enough for a project demo but not enough to draw confident conclusions about the agent's overall reliability.
